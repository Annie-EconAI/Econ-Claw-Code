## 版本一：自然随笔风

### 标题
读博之后才发现，最缺的不是 idea，是整块的思考时间

### 正文

最近越来越有一个感受：做研究最累的部分，其实不是"想不出来"，而是根本没有时间去想。

一天下来，调代码、搭模板、翻旧项目、查格式、改编译错误……每一件都不难，但加在一起就把你的注意力切得稀碎。等到终于能坐下来想想研究问题的时候，已经是晚上十一点半了。

之前看到一句话说得特别准：教授的注意力是被保护的——学生把数据整理好了才汇报上去，PPT 改了五遍才敢 present。但年轻研究者呢，什么都得自己来。

后来偶然看到 GitHub 上一个开源工具 Econ-Claw-Code，试了一下发现还挺实用。它做的事情很简单：你用一句话告诉它你想做什么研究，它帮你匹配方法、调出代码模板、做投稿前检查。

比如你输入"用 DID 估计最低工资对就业的影响"，它直接告诉你可以用哪些命令。需要 event study 模板？一条命令输出，Stata R Python 都有。投稿前想查查有没有 XXX 没删？它也能扫。

说实话不是什么神奇的东西，但确实能省掉不少重复劳动。省下来的那一两个小时，哪怕只是安安静静想一下"这个系数到底说明了什么"，都比再搭一遍回归模板有价值。

做研究最贵的资源是注意力，别把它浪费在复制粘贴上。

🔗 GitHub 搜 Econ-Claw-Code（链接见评论）
零依赖 Python 3.10+ 开箱即用

## 版本二：正经推荐风

### 标题
安利一个社科实证研究的开源工具，真的很实用

### 正文

最近发现一个专门给经济学和社科研究者做的命令行工具，叫 Econ-Claw-Code，开源免费的，试用之后觉得值得推荐。

📌 它能做什么：

① 方法匹配
输入一句研究任务描述（比如"estimate treatment effect using DiD"），自动匹配最相关的计量方法。内置 37 个命令，覆盖 DID、交错DID、IV、RDD、事件研究、合成控制、聚束估计等主流方法。

② 代码模板
11 个发表级模板，Stata / R / Python 三种语言。reghdfe 主回归、event study、平衡性检验表、HonestDiD、Callaway-Sant'Anna、Romano-Wolf……一条命令调出，改变量名就能跑。

③ 投稿前检查
自动扫描占位符（XXX/TODO）、因果语言过度表述、引文缺失。还有数字核验、聚类检查、平行趋势验证等 23 个工具。

④ 全流程覆盖
把研究分成 data → analysis → viz → write → review → submit 六个阶段，每个阶段都有对应的命令和工具，输入需求自动识别你在哪个阶段。

📌 适合谁：
本科生写论文 / 硕博赶 paper / RA 搭 replication package / 教职管多个项目 / 政策研究者

📌 怎么用：
```
git clone https://github.com/Annie-EconAI/Econ-Claw-Code.git
cd Econ-Claw-Code
python -m src.main route "your research question"
```
零依赖，Python 3.10+ 直接跑，不用装任何包。

📌 开源地址：
GitHub 搜 Econ-Claw-Code（评论区放链接）

个人觉得特别适合没有团队、什么都自己做的研究者。不是帮你写论文，是帮你把最机械的那部分工作先接过去，省下来的时间留给真正的思考。
